# Divadlo 3+KK

Website for [Divadlo 3+KK](https://www.tripluskk.com) — built with Next.js (Pages Router), Sanity CMS, and deployed to GitHub Pages.

## Development

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Stack

- **Next.js** (Pages Router) — static export for GitHub Pages
- **Sanity** — CMS for content (projects, about, contact)
- **next-intl** — Czech / English i18n via `[locale]` URL segment
- **next-sitemap** — sitemap generation at build time

---

## Deploying to GitHub Pages

GitHub Pages is a **static host** — no Node.js server. This means:

- `getServerSideProps` → not supported
- `revalidate` (ISR) → not supported
- API routes → not supported
- Next.js built-in `i18n` routing → not supported (requires server redirects)
- `images.remotePatterns` → replace with `images.unoptimized: true`

### `next.config.ts`

Do NOT use `static_site_generator: next` in the workflow — it generates a CommonJS `next.config.js` which breaks if `package.json` has `"type": "module"`.

Instead, configure everything manually using a `GITHUB_PAGES` env variable:

```ts
import type {NextConfig} from 'next';

const isGithubPages = process.env.GITHUB_PAGES === 'true';

const nextConfig: NextConfig = {
    ...(isGithubPages && {
        output: 'export',
        // basePath: '/your-repo-name',  // add if NOT using a custom domain
        images: {
            unoptimized: true,
        },
    }),
    ...(!isGithubPages && {
        images: {
            remotePatterns: [
                {
                    protocol: 'https',
                    hostname: 'cdn.sanity.io',
                },
            ],
        },
    }),
};

export default nextConfig;
```

> **basePath**: Only needed if the site is served at `username.github.io/repo-name` (no custom domain). Set it to `'/your-repo-name'`.

### i18n: `[locale]` URL segment instead of Next.js built-in i18n

Next.js built-in `i18n` config requires a server to handle locale redirects. Remove it entirely and encode the locale directly in the URL path.

**Directory structure:**

```
pages/
  index.tsx               ← redirects to /cs
  404.tsx
  [locale]/
    index.tsx
    about.tsx
    contact.tsx
    projects.tsx
    projects/
      [slug].tsx
  api/
    revalidate.js         ← server-only, removed before GitHub Pages build
  sitemap.xml.jsx         ← server-only, removed before GitHub Pages build; sitemap generated by next-sitemap instead
```

**Root redirect (`pages/index.tsx`):**

```tsx
import {useEffect} from 'react';
import {useRouter} from 'next/router';
import Head from 'next/head';

export default function RootRedirect() {
    const router = useRouter();

    useEffect(() => {
        router.replace('/cs');
    }, [router]);

    return (
        <Head>
            <meta httpEquiv="refresh" content="0;url=cs"/>
        </Head>
    );
}

export function getStaticProps() {
    return {props: {}};
}
```

**Every locale page (`pages/[locale]/about.tsx` etc.):**

```tsx
import {GetStaticPropsContext} from 'next';

export function getStaticPaths() {
    return {
        paths: [
            {params: {locale: 'cs'}},
            {params: {locale: 'en'}},
        ],
        fallback: false,
    };
}

export async function getStaticProps(context: GetStaticPropsContext) {
    const locale = context.params!.locale as string;

    return {
        props: {
            messages: (await import(`../../public/locales/${locale}.json`)).default,
        },
        ...(!process.env.GITHUB_PAGES && {revalidate: 172800}),
    };
}
```

**Dynamic slug page (`pages/[locale]/projects/[slug].tsx`):**

```tsx
export async function getStaticPaths() {
    const slugs = await fetchAllSlugs();
    const paths = slugs.flatMap((slug: string) =>
        ['cs', 'en'].map((locale) => ({
            params: {locale, slug},
        }))
    );
    return {
        paths,
        fallback: process.env.GITHUB_PAGES ? false : 'blocking',
    };
}
```

**`useLocale` hook** — replace `router.locale` with `router.query.locale`:

```ts
import {useRouter} from 'next/router';

export const useLocale = (): 'cs' | 'en' => {
    const router = useRouter();
    const locale = router.query.locale;
    if (locale === 'en') return 'en';
    return 'cs';
};
```

**`_app.tsx` — NextIntlClientProvider:**

```tsx
<NextIntlClientProvider
    locale={(router.query.locale as string) ?? 'cs'}
    messages={pageProps.messages}
>
```

**Navigation links** must include the locale prefix:

```tsx
const locale = useLocale();

<Link href={`/${locale}`}>...</Link>
<Link href={`/${locale}/projects`}>...</Link>

const otherLocale = locale === 'cs' ? 'en' : 'cs';
const switchPath = router.asPath.replace(/^\/(cs|en)/, `/${otherLocale}`);
<Link href={switchPath}>{locale === 'cs' ? 'En' : 'Cz'}</Link>
```

### Remove server-only pages before build

Pages using `getServerSideProps` and API routes must be deleted before the static export. Do this in the workflow, not in code — so they still work in non-GitHub Pages deployments.

- `pages/sitemap.xml.jsx` — uses `getServerSideProps`
- `pages/api/revalidate.js` — API route

### Sitemap via next-sitemap

Since `pages/sitemap.xml.jsx` is removed before the GitHub Pages build, generate the sitemap using `next-sitemap` instead.

**`next-sitemap.config.js`:**

```js
/** @type {import('next-sitemap').IConfig} */
const config = {
  siteUrl: "https://www.your-domain.com",
  generateRobotsTxt: false, // keep your existing public/robots.txt
  outDir: "./out",          // static export output directory
};

export default config
```

**`package.json`:**

```json
"scripts": {
  "build": "next build",
  "postbuild": "next-sitemap"
}
```

The `postbuild` script runs automatically after `npm run build` locally. In the GitHub Actions workflow, call `next-sitemap` explicitly as a separate step (since the workflow calls `next build` directly).

### GitHub Actions workflow

```yaml
name: Deploy Next.js site to Pages

on:
  push:
    branches: ["master"]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Detect package manager
        id: detect-package-manager
        run: |
          if [ -f "${{ github.workspace }}/yarn.lock" ]; then
            echo "manager=yarn" >> $GITHUB_OUTPUT
            echo "command=install" >> $GITHUB_OUTPUT
            echo "runner=yarn" >> $GITHUB_OUTPUT
          else
            echo "manager=npm" >> $GITHUB_OUTPUT
            echo "command=ci" >> $GITHUB_OUTPUT
            echo "runner=npx --no-install" >> $GITHUB_OUTPUT
          fi

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: ${{ steps.detect-package-manager.outputs.manager }}

      # Do NOT use static_site_generator: next — it generates a CommonJS
      # next.config.js which breaks with "type": "module" in package.json
      - name: Setup Pages
        uses: actions/configure-pages@v5

      - name: Restore cache
        uses: actions/cache@v4
        with:
          path: .next/cache
          key: ${{ runner.os }}-nextjs-${{ hashFiles('**/package-lock.json', '**/yarn.lock') }}-${{ hashFiles('**.[jt]s', '**.[jt]sx') }}
          restore-keys: |
            ${{ runner.os }}-nextjs-${{ hashFiles('**/package-lock.json', '**/yarn.lock') }}-

      - name: Install dependencies
        run: ${{ steps.detect-package-manager.outputs.manager }} ${{ steps.detect-package-manager.outputs.command }}

      - name: Remove server-only pages
        run: rm -f pages/sitemap.xml.jsx pages/api/revalidate.js

      - name: Build with Next.js
        run: ${{ steps.detect-package-manager.outputs.runner }} next build
        env:
          GITHUB_PAGES: true

      - name: Generate sitemap
        run: ${{ steps.detect-package-manager.outputs.runner }} next-sitemap

      # Required to prevent GitHub Pages from processing files with Jekyll
      - name: Add .nojekyll
        run: touch ./out/.nojekyll

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./out

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v5
```

### GitHub repository settings

1. Go to **Settings → Pages**
2. Set **Source** to `GitHub Actions`

### Pitfalls

| Problem | Cause | Fix |
|---|---|---|
| `module is not defined in ES module scope` | `configure-pages` with `static_site_generator: next` generates CommonJS config | Remove `static_site_generator: next` from the workflow |
| `Invalid locale returned from getStaticPaths` | `getStaticPaths` returns `locale` key but i18n is disabled | Remove i18n from next.config; use `[locale]` URL param instead |
| `getServerSideProps can not be exported` | Sitemap or other pages use SSR | Delete those pages before build in the workflow; use `next-sitemap` for sitemap generation |
| `fallback: 'blocking'` error | ISR fallback unsupported in static export | Use `fallback: process.env.GITHUB_PAGES ? false : 'blocking'` |
| `revalidate` error | ISR unsupported in static export | Disable with `...(!process.env.GITHUB_PAGES && { revalidate: N })` |
| Images broken | Remote image optimization requires a server | Use `images: { unoptimized: true }` for GitHub Pages builds |
| Links broken with basePath | Internal links don't include basePath automatically in some cases | Verify all `<Link>` hrefs — Next.js should prepend basePath automatically |
